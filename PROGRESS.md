## Phase 2 — UX · 도메인 모델 (완료)

### 워딩 확정

| 용어 | UI 표시 | 영문 도메인 |
|---|---|---|
| 컨테이너 | 가방 | Pack |
| 개별 아이템 | 챙길 것 | Item |
| 출발 트리거 | 떠날 때 | departure |
| 도착 트리거 | 도착할 때 | arrival |

### 도메인 모델

```
users: id, email, display_name, avatar_url, subscription_tier, created_at
packs: id, user_id, name, emoji, color, sort_order, created_at, updated_at
triggers: id, pack_id, type(departure|arrival), lat, lng, radius_meters, label, is_active, created_at
items: id, trigger_id, name, emoji, sort_order, created_at
notification_logs: id, user_id, trigger_id, fired_at, acknowledged_at
check_logs: id, notification_log_id, item_id, checked_at
```

> ⚠️ 미결: items.trigger_id(트리거 종속) vs 글로벌 풀 + 담기 M:N — 와이어프레임은 글로벌 풀 방식으로 설계돼 있어 충돌. hi-fi 전 결정 필요.

### 탭 구조

```
내 가방 | 알림 | 설정
```

### 화면 목록

**내 가방 탭**
- 가방 목록
- 가방 상세 (떠날 때 / 도착할 때 탭)
- 가방 생성 (= 빈 가방 상세)
- 챙길 것 담기 바텀시트
- 위치 설정 (전체화면 지도)

**설정 탭**
- 계정, 권한 상태, 구독 관리, 약관

**탭 외**
| 화면 | 진입 시점 |
|---|---|
| 온보딩 (3스텝) | 최초 실행 |
| 소셜 로그인 | 온보딩 마지막 |
| 권한 요청 | 로그인 직후 |
| 알림 체크리스트 | 출발 감지 알림 탭 시 |
| 페이월 | 가방 3개째 시도 시 |
| 구독 플랜 선택 | 페이월에서 진입 |

### UX 패턴 확정

| 항목 | 결정 |
|---|---|
| 가방 관리 | ··· 케밥 메뉴 → 이름·이모지 변경 / 삭제 |
| 챙길 것 편집 | 아이템 ··· 메뉴 → 편집 바텀시트(이모지 그리드 + 이름) / 가방에서 빼기 |
| 챙길 것 추가 | "챙길 것 담기" 바텀시트 (풀페이지 라이브러리 없음) |
| 새 소지품 생성 | 담기 시트 내 인라인 생성 (이름 = 이모지 검색어) |
| 저장 방식 | 즉시저장, 글로벌 저장 버튼 없음 |
| 드래그 삭제 | 아이템 꾹 누르기 → 드래그, 하단 + 버튼 → 휴지통 드롭존 |
| 위치 설정 | 가방 상세 지도 썸네일 탭 → 전체화면. 반경 떠날 때/도착할 때 공용 |
| 가방 이름·이모지 | [이모지▾][이름 입력] — 이름이 이모지 추천 검색어 역할 겸용 |
| 가방 생성 | 빈 가방 상세와 동일 레이아웃. 하단 CTA "가방 만들기" |
| 가방 리스트 카드 | 이모지 아바타 + 이름/트리거·개수 서브타이틀 + ON/OFF 토글 |
| 담기 시트 트리거 표시 | 떠날 때 / 도착할 때 선택 세그먼트 (어느 트리거에 담는지 명시) |

### 페이월 UX 확정

- **A안(투자심리)** 확정
- 진입 경로 2종 카피 분기:
  - 가방 2개 꽉 찬 경우: "가방 2개가 꽉 찼어요 / 세 번째 차례예요"
  - 빈 가방·다른 진입: "무료는 가방 2개까지 / 세 번째부터 플랜이 필요해요"
- 플랜 선택 모달: 연간 탭 디폴트, 프로 추천 강조, 공통 혜택 카드 하단 분리

### 알림 체크리스트 UX 확정 (C안 — 위치 그리드)

- 잠금화면 푸시에 아이템 미리보기 텍스트 노출
- 푸시 버튼: "다 챙김"(즉시완료) / "열기"(체크리스트 진입)
- 체크리스트: 2열 그리드 + 상단 "전부 챙겼어요" 빠른완료 + N/전체 카운터
- CTA 전체 체크 완료 시 활성화 + "미완료로 닫기" 링크 병행
- ⚠️ 미결: "전부 챙겼어요" 버튼 동작 명확화 필요
